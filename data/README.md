# Data

The raw HLS-CMDS audio archive is **not** committed to this repository (third-party
dataset, and too large for version control). To reproduce the notebook:

1. Obtain `HLS-CMDS.zip` from its original source (IEEE DataPort listing).
2. Place it at `data/raw/HLS-CMDS.zip` relative to the repo root.
3. Run the notebook — it extracts the archive itself and validates the result
   against a fixed expected inventory (535 files: 50 HS, 50 LS, 145 each of
   Mix-H/Mix-L/Mix-M).

## Integrity check

The notebook computes a SHA256 fingerprint of the archive on load (Section 1).
For the version this analysis was run against:

```
2726a714187c5cbbc5ae6b4992fc1995007fb87aecc6ddd2c551d7bf6f5c678d
```

If your download produces a different fingerprint, treat the results in this
notebook as unverified against your copy of the data.

## Derived outputs

Cleaned/derived tables (file manifest, corrected ID mappings, issue log,
experiment results) are checked into [`../outputs/tables`](../outputs/tables) —
these are small, non-audio CSVs generated by the notebook, not a redistribution
of the original audio.
